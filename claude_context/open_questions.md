### Q1: デザイン類似度の閾値設計
**問い**: 「盗作（拒否）」と「フォーク（派生許容）」の境界となるコサイン類似度スコアの閾値はいくつか？

**影響箇所**: `design_ips`の`similarity_vector`検索ロジック / Admin設定画面 / ユーザーへのフィードバックメッセージ

**選択肢案**:
- A. スコア ≥ 0.95 → 拒否 / 0.75 ≤ スコア < 0.95 → フォーク / スコア < 0.75 → 独立IP
- B. 段階的: 最初は閾値を緩め（≥ 0.90で拒否）、UGCが増えた段階で厳しく調整

**提案**: Adminが管理画面から調整できる設計にして初期値をBとし、データが集まったら調整するアプローチ。
**答え**: 提案のとおりで

### Q2: レベニューシェアの多段フォーク分配ロジック
**問い**: AをフォークしたBをフォークしたCが施術された場合、利益はどう分配するか？

**影響箇所**: `design_royalty_nodes`の生成ロジック / `royalty_distributions`の分配計算処理

**現在の仕様書での定義**: 「元祖70%/アレンジャー30%」（2段のみ。3段以降は未定義）

**提案（要承認）**:
```
深さ1（BがAのフォーク）→ Bが施術された場合:
  B（直接デザイナー）: 70%
  A（元祖）: 30%

深さ2（CがBのフォーク。BはAのフォーク）→ Cが施術された場合:
  C（直接デザイナー）: 70%
  B（親）: 21%  ← Bの取り分30%をさらに70/30で分割
  A（祖父）: 9% ← Bが親から受け取る30%の30%

深さ3以降: 分配は3段で打ち切り。プラットフォームが余剰分（端数）を吸収
```

**確認事項**: この分配ロジックで合意するか。端数処理は切り捨てか四捨五入か。
**答え**: 提案のとおりで。端数処理は**1円未満切り捨て**（floor）。余剰分はプラットフォームが吸収。

### Q3: 決済トリガーのタイミング
**問い**: 決済（Stripeキャプチャ）・IP利用料分配はどのイベントで実行するか？

**影響箇所**: `payments.status`の遷移ロジック / `royalty_distributions`の生成タイミング

**選択肢**:
- A. **施術完了確認後**（サロンが「施術完了」ボタン押下後）→ キャプチャ → 分配
  - メリット: トラブル時の返金が容易。ユーザー保護が強い
  - デメリット: サロンがボタンを押し忘れると入金が遅延
- B. **予約確定時**→ 即時キャプチャ → 施術完了で分配
  - メリット: サロンへの入金が早い
  - デメリット: キャンセル時の返金処理が複雑

**提案**: A（施術完了確認後）を推奨。サロンへのリマインド通知で遅延を防止。
**答え**: 提案のとおりで

### Q4: プラットフォーム手数料率と IP 利用料率
**問い**: 以下の料率を確定させたい。

| 項目 | 案 | 根拠 |
|------|----|------|
| プラットフォーム手数料 | 15% | 競合予約サービス（HOT PEPPER: 20〜30%）より安く設定し加盟店獲得を優先 |
| IP利用料（総額比） | 5% | ユーザーへのインセンティブとして十分な額を確保しつつ、サロン負担を抑える |

**全体の金銭フロー例（施術代 ¥8,000の場合）**:
```
ユーザー支払い総額: ¥8,000
  → プラットフォーム手数料 (15%): ¥1,200
  → IP利用料 (5%): ¥400
  → サロン取り分: ¥6,400

IP利用料 ¥400 の分配（フォーク1段の例）:
  → 直接デザイナー (70%): ¥280
  → 元祖デザイナー (30%): ¥120
```

---
**答え**: 提案のとおりでOK

### Q5: サロンの稼働率データの取得方法
**問い**: 「ダイナミック割引」の根拠となる稼働率データはどう取得するか？

**選択肢**:
- A. **サロン自己入力**: 入札フォームで「割引理由」を自由テキスト入力。稼働率はシステムが保持しない
- B. **本プラットフォーム内の予約から自動算出**: プラットフォーム経由の予約だけで稼働率を計算
- C. **外部予約APIとのポーリング連携**: HOT PEPPER等との連携（技術的難易度高・交渉コスト大）

**提案**: Phase 1はA（自己入力）。プラットフォームの予約が増えた段階でBに移行。Cはフェーズ3以降。
**答え**: 提案のとおりで
