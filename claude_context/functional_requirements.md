# 機能要件定義書（Functional Requirements）

## 凡例
- **優先度**: P0=MVP必須 / P1=初期リリース / P2=フェーズ2以降
- **Moat寄与**: ★★★=コアMoat / ★★=Moat補完 / ★=標準機能

## 非機能要件

| 項目 | 要件 |
|------|------|
| API応答速度 | 通常エンドポイント: 200ms以内（p95）|
| AR処理 | リアルタイム30fps以上（DeepAR標準）|
| 可用性 | 99.9%（月間ダウンタイム43分以内）|
| スケーラビリティ | EKSオートスケーリング。逆オークション入札ピーク時を考慮 |
| データ保持 | AR試着画像: 90日（ユーザー設定で延長可）/ 施術カルテ: 永続 |
| 多言語 | 初期は日本語のみ。i18n構造は最初から組み込む |
| アクセシビリティ | WCAG 2.1 AA準拠（性別フリーUIの観点から）|
